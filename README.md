# tableau-book

태블로 기반 데이터 시각화 책을 WikiDocs 형태로 집필하기 위한 리포지토리입니다.

현재 목차는 기초, 시각화, 계산, 데이터 준비, 대시보드 구축까지 학습 흐름이 이어지도록 구성되어 있습니다.

## 목적

- Tableau의 기본 조작부터 데이터 시각화, 계산, 데이터 준비, 대시보드 공유까지 체계적으로 설명합니다.
- 단순 기능 나열이 아니라 실습 가능한 흐름으로 내용을 정리합니다.
- WikiDocs 반영을 고려해 목차와 페이지 네이밍을 안정적으로 유지합니다.

## 현재 구조

```text
.
├── README.md
├── TOC.md
├── assets/
│   └── book-cover.png
├── examples/
└── pages/
    ├── 01-tableau-basics.md
    ├── 01-1-bi-overview.md
    ├── ...
    ├── 05-dashboard-build-and-share.md
    └── 05-5-tableau-cloud-sharing.md
```

## 작성 규칙

- 모든 본문 문서는 `pages/` 아래에 둡니다.
- 파일명은 `NN-topic-name.md` 형식으로 관리합니다.
- 문서 제목도 `NN. 제목` 형식으로 맞춰 WikiDocs 정렬 문제를 방지합니다.
- 이미지와 도식은 `assets/`에 저장합니다.
- 예제 데이터 설명, SQL, 계산식 정리 등은 `examples/`에 둡니다.

## 표지 자산

- 현재 책 표지 기준 파일은 `assets/book-cover.png`입니다.
- 표지를 교체할 때는 같은 파일명을 유지하면 문서와 연결 경로를 바꾸지 않아도 됩니다.
- 표지 원본을 별도로 관리해야 하면 `assets/source/` 같은 하위 폴더를 추가해도 됩니다.

## 작업 흐름

1. `TOC.md`에서 책의 흐름을 먼저 고정합니다.
2. 각 챕터 소개 페이지와 세부 절 원고를 `pages/`에 작성합니다.
3. 이미지가 필요하면 `assets/`에 저장한 뒤 상대 경로로 연결합니다.
4. 초안 단계에서는 완결성보다 구조 일관성을 우선합니다.

## 문서 작성 권장 템플릿

각 절은 아래 흐름을 기본으로 두면 집필 품질이 안정적입니다.

1. 왜 이 주제가 중요한지
2. 핵심 개념과 작동 원리
3. Tableau에서 어떻게 구현하는지
4. 실무에서 자주 생기는 문제
5. 대안 또는 확장 방법
