## 1. Tableau Pulse란?

Tableau Pulse는 핵심 지표를 중심으로 변화, 이상 징후, 추세를 자동으로 요약해서 보여주는 지표 중심 경험입니다.  
공식 자료에서는 작업 흐름 안으로 직접 들어오는 개인화된 인사이트 경험으로 설명합니다.

즉, Pulse는 “사용자가 직접 차트를 조작하며 탐색하는 화면”이라기보다, `중요한 지표가 어떻게 변하고 있는지 먼저 알려주는 기능`에 가깝습니다.

## 2. Pulse의 핵심 기능

공식 문서 기준으로 Pulse는 다음 기능에 강점이 있습니다.

- 지표(metric) 중심 구독
- 이메일, Slack, 모바일 알림
- 자연어 기반 요약
- 이상 추세 및 임계값 경고
- 관련 질문 추천

특히 Discover(Enhanced Q&A)는 여러 지표를 묶어서 함께 탐색하는 기능으로 소개되며, 공식 도움말 기준 Tableau+ 기능입니다.

## 3. 대시보드와 무엇이 다른가?

| 구분 | Tableau Dashboard | Tableau Pulse |
| --- | --- | --- |
| 중심 단위 | 시트와 레이아웃 | 지표(Metric) |
| 사용자 행동 | 직접 탐색 | 자동 요약 확인 |
| 주 용도 | 상세 분석, 비교, 상호작용 | 변화 감지, 빠른 상황 파악 |
| 적합한 질문 | 왜 이렇게 됐는가? | 지금 무엇이 달라졌는가? |

즉, 대시보드는 탐색형이고, Pulse는 모니터링형이라고 보시면 됩니다.

## 4. 실무에서 언제 유용한가?

Pulse는 다음 같은 상황에서 특히 유용합니다.

- 경영진이나 현업 사용자가 매일 핵심 KPI를 확인해야 할 때
- 사용자가 직접 대시보드를 열어보지 않아도 변화 알림을 받아야 할 때
- 매출, 트래픽, 전환율, 만족도처럼 시계열 지표를 지속적으로 추적할 때

반대로 세부 레코드 수준 탐색이나 복잡한 다차원 비교는 Pulse보다 대시보드가 더 적합합니다.

## 5. 주의할 점

공식 도움말에서도 Discover가 특히 잘 맞는 데이터 구조를 따로 설명합니다.

잘 맞는 경우:

- 시계열 데이터
- 집계 가능한 핵심 지표
- 지역, 제품, 세그먼트 같은 명확한 차원

덜 적합한 경우:

- 지나치게 세밀한 로그 데이터
- 시점이 하나뿐인 정적 데이터
- 비정형 텍스트 중심 데이터

즉, Pulse는 “모든 데이터를 AI로 설명해 주는 도구”가 아니라, `구조화된 KPI 데이터를 빠르게 이해하게 해 주는 경험`에 더 가깝습니다.
