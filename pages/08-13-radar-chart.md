## 학습 목표

- 레이더 차트의 개념과 활용 목적을 이해합니다.
- 여러 지표의 균형과 패턴을 비교하는 방법을 설명할 수 있습니다.
- Tableau에서 계산된 필드와 경로 마크를 활용해 레이더 차트를 구현하는 원리를 이해합니다.

## 1. 레이더 차트란?

레이더 차트는 여러 지표를 방사형 축으로 배치하고, 값을 연결해 다각형 형태로 표현하여 항목별 성과 균형과 패턴을 비교하는 차트입니다.

- 각 축은 하나의 지표를 의미합니다.
- 중심에서 멀수록 값이 크다고 해석합니다.
- 여러 항목을 같은 축 구조 위에서 비교할 수 있습니다.

즉, 레이더 차트는 개별 수치보다 `지표의 균형과 모양`을 읽는 데 적합합니다.

## 2. 레이더 차트를 자주 쓰는 이유

레이더 차트는 하나의 대상이 여러 항목에서 어떤 강점과 약점을 가지는지 직관적으로 보여줄 수 있습니다.

대표적인 활용 예시는 다음과 같습니다.

- 제품 성능 지표 비교
- 조직 역량 평가 시각화
- 브랜드별 평가 항목 비교

실무에서는 다음 질문에 답할 때 유용합니다.

- 특정 대상이 어떤 항목에서 강한가?
- 여러 대상의 패턴이 비슷한가, 다른가?
- 전반적으로 균형 잡힌 성과를 보이는가?

즉, 레이더 차트는 `총점`보다 `구성 패턴`을 읽는 데 강합니다.

## 3. Tableau에서 레이더 차트 만드는 방법

Tableau에서는 극좌표 변환을 위한 계산된 필드와 경로(Path) 마크를 활용해 레이더 차트를 구현합니다.

핵심 원리는 다음과 같습니다.

1. 각 지표를 각도(Angle)로 배치합니다.
2. 각 지표 값을 반지름 값처럼 사용합니다.
3. 삼각함수로 `X`, `Y` 좌표를 계산합니다.
4. 경로(Path)로 점들을 연결해 다각형을 만듭니다.

즉, 레이더 차트는 Tableau 기본 차트 메뉴에서 바로 선택하는 방식이 아니라, 좌표 변환 계산을 통해 구현하는 커스텀 차트입니다.

실무적으로 중요한 포인트는 다음과 같습니다.

- 축 범위를 동일하게 맞추지 않으면 비교 해석이 왜곡될 수 있습니다.
- 지표 간 단위가 다르면 정규화가 필요합니다.
- 너무 많은 축을 넣으면 가독성이 급격히 떨어집니다.

그래서 레이더 차트는 `스케일 정규화`가 핵심입니다.  
축이 제각각이면 모양 차이가 실제 성과 차이가 아니라 스케일 차이로 보일 수 있기 때문입니다.

실무에서는 두 가지 방식으로 구현할 수 있습니다.

### 1. 계산된 필드 기반 직접 구현

- 자유도가 높습니다.
- 좌표 계산과 Path 구성이 필요합니다.
- 구현 난이도는 높은 편입니다.

### 2. 비주얼리제이션 확장 프로그램 활용

- 빠르게 만들 수 있습니다.
- 구현 부담이 적습니다.
- 다만 자유도는 제한될 수 있습니다.

즉, 레이더 차트의 핵심은 `방사형 구조 자체보다도 스케일을 어떻게 맞추는가`에 있습니다.
